# CUDA Batch Image Processing Pipeline

GPU-accelerated batch image processing using the **CUDA NPP** (NVIDIA Performance Primitives) library. Processes hundreds of images through a 3-stage pipeline:

1. **RGB → Grayscale** — `nppiRGBToGray_8u_C3C1R`
2. **Gaussian Blur** — `nppiFilterGauss_8u_C1R` (configurable kernel: 3×3, 5×5, 7×7)
3. **Sobel Edge Detection** — `nppiFilterSobelHorizBorder_8u_C1R` + `nppiFilterSobelVertBorder_8u_C1R`

All stages execute on the GPU with zero CPU involvement in the pixel math.

```bash
cd cuda-batch-imgproc

# 2. Fetch stb headers + sample images + build + run (all-in-one)
chmod +x run.sh scripts/*.sh
./run.sh

# Or step by step:
make setup                    # fetch stb headers
bash scripts/download_images.sh  # populate data/input/ with 100 images
make all                      # compile -> bin/cuda_imgproc
make run                      # execute pipeline
```

## Algorithm Details

### Stage 1 — RGB to Grayscale
Uses the ITU-R BT.601 luminance formula via `nppiRGBToGray_8u_C3C1R`:
```
Y = 0.299·R + 0.587·G + 0.114·B
```

### Stage 2 — Gaussian Blur
`nppiFilterGauss_8u_C1R` applies a separable Gaussian kernel of configurable size (3×3, 5×5, or 7×7) to reduce noise before edge detection.

### Stage 3 — Sobel Edge Detection
Horizontal and vertical Sobel filters are applied separately using border-replicated variants (`nppiFilterSobelHorizBorder_8u_C1R`, `nppiFilterSobelVertBorder_8u_C1R`), then combined with saturating addition (`nppiAdd_8u_C1RSfs`) to produce an edge magnitude map.

### Memory Management
All GPU allocations use a `DeviceBuffer` RAII wrapper (see `include/utils.h`) that guarantees `cudaFree` on scope exit, preventing leaks across the per-image loop.

## Lessons Learned

- NPP border modes (`NPP_BORDER_REPLICATE`) are required for Sobel variants that take a source ROI offset; the non-border versions don't exist for all filter types.
- `stb_image` loads images as RGB by forcing `desired_channels=3`, which simplifies the NPP `C3` input requirement.
- Saturating add (`nppiAdd_8u_C1RSfs` with scale=0) clips combined Sobel responses naturally at 255, giving a clean uint8 edge map without extra normalization.
- Colab T4 shows ~1–3 ms per 512×512 image including H2D/D2H transfers, dominated by PCI-e bandwidth at small sizes; larger images (2048×2048) better utilize NPP's parallelism.
